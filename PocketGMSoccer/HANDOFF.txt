WHAT THIS IS
------------
A football-manager game. Single-file React app (JSX transpiled in-page by a
vendored Babel), bundled offline inside a SwiftUI WKWebView iOS app — same
pattern as the baseball app in the same repo.

WHERE THE CODE LIVES
--------------------
- Repo:    kyleb0226/pocketgm-baseball-ios  (this is where I can push)
- Branch:  claude/soccer-game-variant-l0yn8l
- Game:    PocketGMSoccer/Web/index.html      <-- the whole game is this one file
- iOS app: PocketGMSoccer/  (ContentView.swift, PocketGMSoccerApp.swift, Web/)
- Xcode:   PocketGMSoccer.xcodeproj
- Related: kyleb0226/soccer-gm = intended home for the single-file WEB build
           (CDN scripts, like kyleb0226/baseball-gm). Not yet populated — needs
           a multi-repo session OR copy the web build over by hand.

HOW TO RUN IT
-------------
- Mac (no setup):   open PocketGMSoccer/Web/index.html in a browser.
- iPhone (native):  open PocketGMSoccer.xcodeproj in Xcode, Run to device.
- Anywhere (web):   GitHub Pages workflow already in repo (.github/workflows).

HOW TO VALIDATE CHANGES (do this before committing — no iOS build needed)
------------------------------------------------------------------------
1) Parse check: transpile the #app-src JSX with the vendored Babel under Node
   and confirm it parses (catches syntax errors).
2) Engine check: strip the React mount, expose engine fns, run several full
   seasons headless; assert every league is balanced (W==L, GF==GA, 38 games),
   champions/cups/awards/transfers work, player count stays stable.
3) Render check: jsdom + the vendored React/ReactDOM, mount <App/>, click
   through every tab + play a matchday, assert zero console errors.
   (jsdom must be npm-installed in a scratch dir; set url:'https://game.local/'
   so localStorage works; load react UMD then set global.React BEFORE react-dom;
   React 18 render is async so flush a few setTimeout(0) ticks before asserting.)
